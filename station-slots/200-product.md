# Fallback Station 200 -- Product / BA

Use this fallback only when the learner has no Module 200 Final Kata spec.

**Station source:** `fallback`

- **Reads:** `runs/<feature-slug>/100-opportunity-brief.md`
- **Writes:** `runs/<feature-slug>/200-spec.md`

## Instructions

Write a feature spec with user story, in-scope and out-of-scope bullets, acceptance criteria,
non-functional constraints, dependencies, open questions, and a handoff note for Design.

## Human gates

Pause if priority, scope, compliance-visible behavior, or acceptance criteria require owner
approval.

## Done when

Design can draft states and flows, and QA can later derive tests from the same criteria.
