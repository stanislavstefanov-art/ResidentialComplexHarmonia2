# Runs

Each factory run writes here under `runs/<feature-slug>/`:

- `100-opportunity-brief.md`, `200-spec.md`, `300-design.md`, `400-architecture.md`,
  `500-implementation.md`, `700-data-design.md`, `800-infra.md`, `900-security-review.md`,
  `600-test-plan.md`, `1000-release-plan.md` ‚Äî the station output chain, in station order
- `transcript.md` ‚Äî the running log of the line
- `seam-ledger.md` ‚Äî at least 3 seam findings
- `human-gates.md` ‚Äî at least 2 human gates
- `eval-report.md` ‚Äî pass evidence
- `cost-log.md` ‚Äî model/cost/run-boundary note
- `risk-note.md` ‚Äî top risks and residual risk
- `final-recommendation.md` ‚Äî adopt / pilot / defer recommendation
- `run-record.md` ‚Äî your filled-in copy of [`run-record.template.md`](run-record.template.md)
- `sdlc-factory-adaptation-note.md` ‚Äî optional Lane 2 comparison if attempted

`<feature-slug>` is a short name for the feature you ran (for example, `checkout-discount`).
